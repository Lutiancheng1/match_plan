# 短片段 Observation 标注手册

> 更新时间：2026-03-27  
> 适用范围：`06_training_pool/02_clip_observation`

## 1. 目标

短片段标注的目标，是让模型先学会看懂动态足球事实，而不是直接下规则结论。

当前最重要的是这四类：

- 重接触 / 危险犯规候选
- 倒地 / 伤停 / 医疗暂停候选
- 出界 / 底线候选
- 角球 / 球门球候选

## 2. 当前课程目标

### `steady_reference`

用途：

- 给模型稳定的 live 参考画面
- 用来对比高价值事件片段

### `contact_restart_scan`

用途：

- 当前最优先复核的课程片段
- 重点检查：
  - 有没有重接触
  - 有没有倒地或停表候选
  - 有没有出界 / 底线 / 角球候选

### `manual_visual_review`

用途：

- 当前还不能自动归类的视觉候选
- 先人工描述事实，不急着给规则或交易判断

## 3. 标注顺序

每个 clip 统一按下面顺序看：

1. 先判断是不是 live 还是 replay
2. 再描述连续动作里发生了什么
3. 再标 `event_candidates`
4. 最后才写一句 `explanation_short`

## 4. 标注原则

- clip 层只标“视觉事实”和“事件候选”
- 角球、球门球、红牌这类结果，优先放到 rule teaching 再下结论
- 看不清最后触球方时，不要硬断 `corner`
- 看不清接触点时，不要硬断 `heavy_contact_foul`

## 5. 高优先级 clip 怎么看

如果 `review_priority=high`，至少要回答：

- 有没有明显的重接触
- 有没有球员倒地或治疗候选
- 球是否可能已经出界
- 如果接近底线，是否值得继续做规则映射

## 6. 当前不做

- 不在 clip 层直接写交易建议
- 不在 clip 层直接写盘口方向
- 不把 replay 片段当成实时事件确认
