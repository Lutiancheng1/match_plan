# 足球知识库与任务定义

> 更新时间：2026-03-27  
> 当前优先级：最高

## 1. 当前最先要解决的问题

现在还不能直接做提醒、对冲和语音层。

最先要做的是让模型真正具备三类知识：

- 看懂足球画面
- 看懂足球规则
- 看懂盘口变化

交易逻辑必须放在这三层之后。

## 2. 知识库拆分

### A. 足球画面知识

- live play / replay / scoreboard / crowd / stoppage
- 普通身体接触 vs 重踩 / 危险犯规
- 球出界、底线、边线、角球候选
- 医疗暂停、换人、争议判罚

### B. 足球规则知识

- 最后触球方与球权
- 边线球、球门球、角球的判定规则
- 红牌、黄牌、换人、伤停的比赛影响

### C. 盘口知识

- 让球、赔率、水位
- 时间流失如何影响盘口
- 什么叫重定价
- 什么叫噪声波动

### D. 交易知识

- 第一腿、第二腿
- 先手窗口、失效窗口
- 对冲与锁利的基本定义

## 3. 当前任务顺序

1. 建立足球知识库和任务定义
2. 统一单帧 / 多帧 observation schema
3. 补高质量教学样本
4. 让大模型先学会“懂球”
5. 再做规则引擎
6. 最后才做提醒、语音、训练蒸馏

## 4. 当前已落地的第一批任务

当前已经开始落地的，只限于“懂球底座”：

- 单帧 observation 候选样本池
- 短片段 observation 候选样本池
- 规则教学 stub 层
- 单帧标注手册

当前还没有开始的：

- 提醒层
- 对冲层
- 语音播报层
- 自动下注层

## 5. 当前最重要的现实限制

不是模型先不够强，而是合格教学素材还不够。

当前有效 Gold 足球样本只覆盖少量比赛，因此 Phase 1 的下一步除了继续人工标注，还必须继续补：

- 更长的官方/合法足球比赛视频
- 更明确包含重接触、伤停、出界、角球的比赛
- 可作为 holdout 的独立验证比赛
