# 足球实时观察模型候选池手册

> 更新时间：2026-03-26  
> 适用范围：足球实时观察、离线复核、教师模型、训练蒸馏

## 1. 角色划分

### 在线实时观察员模型

负责：

- 持续看足球直播帧
- 输出 observation / alert 纯 JSON
- 低延迟、稳定、可长时间运行

### 离线复核模型

负责：

- 困难样本复判
- 半自动打标
- 复核错误案例

### 教师模型

负责：

- 少量高价值样本深度理解
- 边界样本 / 争议样本提纯
- 为后续小模型蒸馏产高质量标签

## 3. 核心 benchmark 指标

- JSON 合法率
- 单帧 / 短片段延迟
- 实时提醒延迟
- 强事件识别准确率
- 角球 / 重启识别准确率
- 长时间运行稳定性

## 4. 默认选择规则

- 在线主力：优先 7B~9B 级里最稳、最适合蒸馏的
- 离线复核：优先 30B~40B 级里最准确的
- 教师模型：只用于高价值样本和边界样本提纯

## 5. 当前推荐组合

- 在线主力：`Qwen3.5-VL-9B-8bit-MLX-CRACK`
- 离线复核：`Qwen3.5-VL-35B-A3B-4bit`
- 教师模型：`Qwen3.5-VL-122B-A10B-4bit`

## 6. 训练路线提醒

- 最终在线跟看直播的模型，不应默认是 35B 本体
- 当前更合理路线是：
  1. 用 35B 做教师
  2. 用现有 pipeline 产训练样本
  3. 蒸馏到 9B (LoRA) 或更小模型
