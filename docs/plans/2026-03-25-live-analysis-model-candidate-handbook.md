# 直播理解模型候选池手册

> 更新时间：2026-03-25  
> 适用范围：本地直播理解、离线复核、样本打标、知识库整理

## 1. 这份文档的目标

这份文档不把路线写死在当前本地已有模型上，而是维护一套“可替换、可扩充、可 benchmark”的候选模型池。

原则：

- 模型只是插槽，不是业务逻辑本体
- 所有模型都必须通过统一 benchmark
- 不因为名气大就直接上生产
- 当前本地模型先作为第一批基线
- 更好的开源模型后续可下载、可接入、可替换

## 2. 模型角色划分

### 2.1 在线主力模型

负责：

- 持续看直播帧
- 输出纯 JSON
- 低风险、稳定、可长时间运行

重点指标：

- JSON 合法率
- 单帧/短片段延迟
- OCR/比分识别准确率
- 长时间运行稳定性

### 2.2 离线复核模型

负责：

- 困难样本复判
- 半自动打标
- 复核错误案例

重点指标：

- 语义理解质量
- 多帧/短视频理解能力
- 对困难样本的稳定性

### 2.3 教师模型

负责：

- 少量高价值样本深度理解
- 边界样本/争议样本复核

重点指标：

- 上限质量
- 不是速度

### 2.4 文本整理模型

负责：

- 标签清洗
- 知识库归纳
- 错误案例总结
- prompt 和 schema 辅助整理

重点指标：

- 文本总结能力
- schema 修复能力
- 低成本批处理能力

## 3. 当前本地已安装模型

### 3.1 Qwen2.5-VL-7B-Instruct-4bit

- 类型：VLM
- 当前状态：已安装
- 建议角色：在线主力基线
- 推荐优先级：高
- 优点：
  - 轻量
  - 更适合先跑通在线直播理解
  - 风险低
- 风险：
  - 对复杂场景和困难样本上限有限

### 3.2 Qwen3.5-VL-35B-A3B-4bit

- 类型：VLM
- 当前状态：已安装
- 建议角色：离线复核主力
- 推荐优先级：高
- 优点：
  - 质量更高
  - 更适合困难样本复判和半自动打标
- 风险：
  - 不适合作为第一版常驻在线主力

### 3.3 Qwen3.5-VL-122B-A10B-4bit

- 类型：VLM
- 当前状态：已安装
- 建议角色：教师模型候选
- 推荐优先级：中
- 优点：
  - 适合少量高价值样本复核
- 风险：
  - 资源开销大
  - 不适合常驻在线

### 3.4 Qwen3.5-27B-4bit

- 类型：文本模型
- 当前状态：已安装
- 建议角色：知识库整理、标签清洗、错误归纳
- 推荐优先级：高
- 风险：
  - 不参与视觉主链

### 3.5 Qwen2.5-Coder-14B-Instruct-4bit

- 类型：文本/代码模型
- 当前状态：已安装
- 建议角色：代码辅助、工具链整理、schema 辅助
- 推荐优先级：中
- 风险：
  - 不作为视觉理解主力

## 4. 建议纳入的外部开源候选

### 4.1 Qwen2.5-VL 系列

参考：
- [Qwen2.5-VL 官方介绍](https://qwenlm.github.io/blog/qwen2.5-vl/)

建议用途：
- 作为当前在线基线的持续参考

### 4.2 InternVL3-8B

参考：
- [InternVL3-8B-hf](https://huggingface.co/OpenGVLab/InternVL3-8B-hf)

建议角色：
- 在线主力替代候选

适用原因：
- 适合作为 8B 级直播理解对照模型
- 可以和 Qwen2.5-VL-7B 做同档位 benchmark

### 4.3 InternVL3-38B

参考：
- [InternVL3-38B-hf](https://huggingface.co/OpenGVLab/InternVL3-38B-hf)

建议角色：
- 离线复核候选

适用原因：
- 适合作为 35B 级替代方案，与当前本地 35B 对比

### 4.4 MiniCPM-V 4.5

参考：
- [MiniCPM-V-4_5](https://huggingface.co/openbmb/MiniCPM-V-4_5)

建议角色：
- 值得优先纳入 benchmark 的视频理解候选

适用原因：
- 官方模型卡强调较强的视频理解
- 对直播场景的记分牌、OCR、多帧理解更值得重点测试

### 4.5 Qwen2.5-Omni

参考：
- [Qwen2.5-Omni 官方介绍](https://qwenlm.github.io/blog/qwen2.5-omni/)

建议角色：
- 第二批候选

适用原因：
- 如果后续想把音频、哨声、解说等也纳入，可考虑加入

## 5. 统一 benchmark 指标

所有候选模型必须跑同一套指标：

- JSON 合法率
- 比分/OCR 准确率
- 比赛时间识别准确率
- 半场/全场状态识别准确率
- 强事件候选召回率
- 长时间连续推理稳定性
- 单帧平均延迟
- 短片段平均延迟
- 内存/显存占用
- 长时间运行是否容易格式漂移

## 6. 默认选择规则

- 在线主力：优先 7B~9B 级里最稳的
- 离线复核：优先 30B~40B 级里最准确的
- 教师模型：只用于高价值样本
- 文本整理：单独选文本模型，不拿视觉模型浪费资源

## 7. 当前推荐默认组合

如果今天就开始第一阶段，默认组合为：

- 在线主力：`Qwen2.5-VL-7B-Instruct-4bit`
- 离线复核：`Qwen3.5-VL-35B-A3B-4bit`
- 教师模型：`Qwen3.5-VL-122B-A10B-4bit`
- 文本整理：`Qwen3.5-27B-4bit`
- 代码辅助：`Qwen2.5-Coder-14B-Instruct-4bit`

## 8. 后续下载策略

不是先下载很多模型，而是：

1. 先用本地现有模型建立 benchmark 基线
2. 再补一个外部 8B 候选
3. 再补一个外部 30B~40B 候选
4. 证明有收益，才继续扩模型库

建议优先下载顺序：

1. `MiniCPM-V 4.5`
2. `InternVL3-8B`
3. `InternVL3-38B`
4. `Qwen2.5-Omni`（仅在后续需要音频时）
