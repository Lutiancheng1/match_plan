# 事件 + 盘口联合评测标准

> 更新时间：2026-03-26  
> 适用范围：Phase 2 联合评测、样本复核、知识库建设

## 1. 目标

第二阶段不是让模型继续泛化描述比赛，而是验证：

- 场内事件是否足以引起盘口重定价
- 重定价方向是否可判断
- 是否存在先手买入价值

## 2. 输入组成

联合评测的输入应包含两部分：

### A. 事件侧输入

- clip 视频 / 关键帧
- 第一阶段强事件标签输出
- scene type
- 比分 / 比赛时间 / 停顿信息

### B. 盘口侧输入

- 当时盘口片段
- 事件发生前后的赔率变化窗口
- 让球线 / 大小球线
- 剩余比赛时间
- 简化后的时间线摘要

## 3. 联合评测要问的问题

模型或规则系统必须回答：

1. 这个事件会不会引起盘口变化？
2. 盘口更可能向哪边变？
3. 是短时波动还是持续重定价？
4. 当前是不是先手买入窗口？
5. 是否值得等待第二腿对冲？

## 4. 统一输出字段

联合评测阶段的输出最少应包含：

- `repricing_expected`
- `repricing_direction`
- `repricing_strength`
- `first_leg_side`
- `first_leg_urgency`
- `hedge_window_expected_sec`
- `edge_rationale_short`

## 5. Ground Truth 参考

联合评测不能只靠主观判断，必须对照实际盘口时间线：

- 事件前赔率
- 事件后 15 秒赔率
- 事件后 30 秒赔率
- 事件后 60 秒赔率

主要看：

- 是否真的发生明显重定价
- 方向是否与判断一致
- 变化幅度是否足够支持套利

## 6. 评测维度

至少评估：

- 方向判断准确率
- 是否值得先手买入判断的准确率
- 联动判断的保守性
- 误报率
- 漏报率

## 7. 进入下一阶段的标准

只有当联合评测已经能稳定回答：

- 哪个事件值得关注
- 哪种盘口变化真正具有交易价值

后面才有意义去做“套利机会评分器”。
