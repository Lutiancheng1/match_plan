# 足球实时套利助手总计划

> 更新时间：2026-03-26  
> 当前唯一主线：足球 `FT`  
> 适用范围：`/Users/niannianshunjing/match_plan/recordings`、`/Users/niannianshunjing/match_plan/live_dashboard`、`/Users/niannianshunjing/match_plan/analysis_vlm`

## 1. 顶层目标

项目的最终目标不是做离线 benchmark 展示，也不是做泛化的“看懂比赛”模型。

最终目标是：

**训练出一个足球实时观察模型，让它跟着我们一起看直播，实时识别关键场内事件，结合盘口、赔率和时间变化，及时给出提醒、建议方向和语音播报。**

当前重新排序后的第一优先不是提醒层，而是知识底座和 observation 层。

第一版系统定位固定为：

- 实时盯盘助手
- 人工套利辅助
- 先懂球，再谈提醒
- 不做自动下注

## 2. 产品形态

最终产品由三层组成：

### Layer 1. 实时观察层

负责看足球直播，提取视觉事实：

- 强事件
- 出界 / 底线 / 角球候选
- 比分、比赛时间、镜头状态

### Layer 2. 规则与联动层

负责把视觉事实映射成：

- 足球规则结果
- 盘口可能的重定价方向
- 当前是否仍有先手窗口

### Layer 3. 训练与后续输出层

负责输出：

- observation 数据
- 规则判断输入
- 后续训练样本

提醒、语音和交易逻辑都属于更后面的阶段。

## 3. 双主线

### 主线 A：强事件重定价线

当前优先级最高，重点是：

- 重踩 / 严重犯规
- 疑似伤退
- 红牌风险
- 医疗暂停
- 比赛末段时间流失

目标是抢“事件发生到盘口完全重定价”之间的窗口。

### 主线 B：角球 / 重启线

第二优先级，重点是：

- 球是否出界
- 边线还是底线
- 最后触球方
- 是否应判角球

这条线更规则化，更适合训练闭环，但工程顺序放在强事件线之后。

## 4. 硬边界

- 球种永久锁定为足球 `FT`
- 只使用“视频可播 + 数据已绑定”的比赛
- 无数据绑定比赛不录、不入库、不评测、不训练
- 第一版不自动下注
- `35B` 级模型只做离线教师、复核和样本提纯，不做最终在线主力

## 5. 阶段路线

### Phase 0. 足球合法素材生产

目标：

- 继续稳定 `recordings + live_dashboard`
- 只生产足球合法素材
- 保证素材、时间线、viewer、分析片段稳定产出

### Phase 1. 足球知识库与任务定义

目标：

- 明确模型到底先学什么
- 先定义足球画面知识、规则知识、盘口知识
- 统一任务边界

### Phase 2. observation schema 与 benchmark 统一

目标：

- 统一单帧、多帧、联合评测三层 JSON
- 保证后续样本、规则和训练都用同一套 contract

### Phase 3. 教学样本建设

目标：

- 补高质量足球教学样本
- 先让模型学会懂球，不先学交易

### Phase 4. 足球实时观察能力

目标：

- 先做足球实时观察员，而不是直接做交易引擎
- 先让系统稳定看到事实

当前优先：

- 强事件 observation
- 比分、时间、scene_type
- replay 风险和可交易性判断

### Phase 5. 事件 / 规则 / 盘口联合评测

目标：

- 验证视觉事实是否足以支持交易提醒
- 判断方向、窗口和时机

### Phase 6. 规则引擎、提醒、语音和蒸馏

目标：

- 在 observation 稳定后再进入提醒和规则层
- 最后才做训练蒸馏
